UNIX 教程第四节

4.1 通配符

字符 * 和 ?

字符 * 称为通配符，将与文件（或目录）名称中零个或多个字符匹配。 例如，在 unixstuff 目录中，输入

% ls list*

这将列出当前目录中以 list.... 开头的所有文件。

尝试输入

% ls *list

这将列出以 ....list 结尾的当前目录中的所有文件。

字符 ？ 将恰好匹配一个字符。
所以 ls ?ouse 会匹配像 house 和 mouse这样的文件，但不会抱怨。
尝试输入

% ls ?list

4.2 文件名约定

我们在此应该注意，目录只是一种特殊类型的文件。 因此，命名文件的规则和约定也适用于目录。

在命名文件中，应避免使用 / * & % 这样具有特殊含义的字符， 另外，避免在名称中使用空格。 命名文件最安全的方法是仅使用字母数字字符，即字母和数字，以及 _（下划线）和 . （点）。

文件名通常以小写字母开头，并且可以以点后跟一组表明文件内容的字母结尾。 例如，所有由 C 代码组成的文件都可以用 .c 结尾命名，例如 prog1.c。 然后，为了列出主目录中包含 C 代码的所有文件，只需在该目录中键入 ls *.c 。

注意：某些应用程序生成的所有输出文件使用相同的名称。

例如，除非给出适当的选项，否则某些编译器会生成名为 a.out 的编译文件。 如果您忘记使用该选项，建议您立即重命名已编译的文件，否则下一个此类文件将覆盖它，而丢失源文件。

4.3 获取帮助

在线手册

有在线手册，提供有关大多数命令的信息。 手册页告诉您哪个特定命令可以采用哪些选项，以及每个选项如何修改命令的行为。 键入man命令以读取特定命令的手册页。

例如，要查找有关 wc （word count）命令的更多信息，请输入

% man wc

另外

% whatis wc

给出了命令的单行描述，但省略了有关选项等的任何信息。

关于

当您不确定命令的确切名称时，

% apropos keyword

将为您提供在手册页头中带 关键字 的命令。 例如，尝试输入

% apropos copy

小结

* 匹配任意数量的字符
? 匹配单个字符
man command 阅读命令的在线手册页
whatis command 命令的简要说明
apropos keyword 在手册页中匹配带有关键字的命令
